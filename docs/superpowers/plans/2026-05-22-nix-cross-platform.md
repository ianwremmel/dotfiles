# Nix Cross-Platform (Linux + macOS) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the Nix/home-manager setup work on Linux (both arches) as well as macOS — multi-system flake, OS-aware `nix` plugin (single-user install on Linux, no daemon), `./apply` Linux branch — and validate a real activation in a Linux container.

**Architecture:** The flake enumerates `[aarch64-darwin x86_64-linux aarch64-linux]` and exposes one `homeConfigurations."<user>@<system>"` per system (username from the untracked `host.nix`). `home.nix` derives the home directory from OS + user. The `nix` plugin installs Nix per-OS (Determinate+daemon on macOS, official `--no-daemon` single-user on Linux), branches detection/profile-sourcing accordingly, passes `--extra-experimental-features` (the official Linux installer doesn't enable flakes), and builds `<whoami>@<currentSystem>`. `./apply` runs the full framework on macOS and only the nix step on Linux.

**Tech Stack:** Bash 5, Nix (flakes, `nix-command`), home-manager, the Determinate and official Nix installers, Docker (Linux test vehicle).

## Task 2: OS-aware `nix` plugin

Rewrite `plugins/nix/nix` to install per-OS (Determinate on macOS, official single-user on Linux), branch detection/profile-sourcing by OS, pass experimental-feature flags, and build the `<user>@<system>` target. Verify on macOS that the plugin still installs-detects, builds, and activates, and that the skip paths still work.

**Files:**
- Modify: `plugins/nix/nix` (full replacement below)

- [ ] **Step 1: Replace `plugins/nix/nix` with this exact content**

```bash
#!/usr/bin/env bash

export DOTFILES_NIX_DEPS=()

# Nix install/profile locations differ by OS: macOS uses the multi-user daemon
# install (Determinate); Linux uses a single-user install (no daemon) whose
# profile script and nix binary live under ~/.nix-profile. Detection and
# profile-sourcing branch accordingly. We detect by path (not just `command -v
# nix`) because the framework runs apply hooks in a non-login subshell where nix
# may not be on PATH even when installed.
_dotfiles_nix_profile_script () {
  if [ "$(uname -s)" = Darwin ]; then
    echo /nix/var/nix/profiles/default/etc/profile.d/nix-daemon.sh
  else
    echo "$HOME/.nix-profile/etc/profile.d/nix.sh"
  fi
}

_dotfiles_nix_is_installed () {
  command -v nix >/dev/null 2>&1 \
    || [ -e /nix/var/nix/profiles/default/bin/nix ] \
    || [ -e "$HOME/.nix-profile/bin/nix" ]
}

_dotfiles_nix_load_profile () {
  local script
  script="$(_dotfiles_nix_profile_script)"
  if ! command -v nix >/dev/null 2>&1 && [ -f "$script" ]; then
    # The profile script may reference unset vars; relax `set -u` while sourcing.
    set +u
    # shellcheck disable=SC1090
    source "$script"
    set -u
  fi
}

_dotfiles_nix_install () {
  if _dotfiles_nix_is_installed; then
    debug 'nix already installed'
    return 0
  fi
  if [ "$(uname -s)" = Darwin ]; then
    log 'Installing Nix (Determinate Systems installer)'
    curl -fsSL https://install.determinate.systems/nix | sh -s -- install --no-confirm
  else
    # Single-user, no daemon. The official installer does not need an init system.
    log 'Installing Nix (official single-user install, no daemon)'
    curl -L https://nixos.org/nix/install | sh -s -- --no-daemon --no-channel-add
  fi
}

dotfiles_nix_apply () {
  if [ "${DOTFILES_NIX_SKIP:-0}" -eq 1 ]; then
    log 'DOTFILES_NIX_SKIP set; skipping nix'
    return 0
  fi

  if [ "${DOTFILES_AIRPLANE_MODE:-0}" -eq 1 ]; then
    log 'Airplane mode; skipping nix (install and build require network)'
    return 0
  fi

  _dotfiles_nix_install
  _dotfiles_nix_load_profile

  if ! command -v nix >/dev/null 2>&1; then
    error 'nix is not available after install; aborting nix plugin'
    return 1
  fi

  # The official Linux installer does not enable flakes by default, so pass the
  # experimental features explicitly. Harmless where already enabled (macOS).
  local -a nixflags=(--extra-experimental-features 'nix-command flakes')

  # Host-specific values (currently just the username) live in an untracked,
  # plugin-generated nix/host.nix so they stay out of git.
  printf '# Generated by the nix plugin — host-specific, not tracked in git.\n{ username = "%s"; }\n' \
    "$(whoami)" > "$DOTFILES_ROOT_DIR/nix/host.nix"

  # The flake exposes one homeConfigurations."<user>@<system>" per supported
  # system; build the one matching this machine.
  local system target
  system="$(nix "${nixflags[@]}" eval --impure --raw --expr 'builtins.currentSystem')"
  target="$(whoami)@$system"

  local tmpdir
  tmpdir="$(mktemp -d)"

  log "Building home-manager configuration ($target)"
  nix "${nixflags[@]}" build \
    "path:$DOTFILES_ROOT_DIR/nix#homeConfigurations.\"$target\".activationPackage" \
    --out-link "$tmpdir/result"

  log 'Activating home-manager configuration'
  "$tmpdir/result/activate"

  # Clean up on the success path. A failed build/activate aborts under `set -e`
  # before this and leaks one small temp dir, which is an acceptable rare case.
  rm -rf "$tmpdir"
}
```
